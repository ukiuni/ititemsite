# ClawdBot向けプロンプト

以下の要件で「IT関連アイテム紹介サイト」を自動生成してください。GitHub Actions + GitHub Pagesで公開する前提です。

## 目的
- Zenn / note / Qiita からAPIで記事を取得し、IT関連アイテム（椅子・机・モニター・マウス・キーボード・服・靴・PC・パーツ等）を抽出する。
- 抽出したアイテムの紹介ページを自動生成する。
- Amazonアソシエイトで収益化できる紹介リンクを必ず付与する（アソシエイトID: `ukiuni-22`）。

## 必須仕様
1. **記事取得**
   - Zenn / Qiita / note からAPIで記事を取得する。
   - 取得失敗時は既存のキャッシュ（リポジトリにコミット済み）を使用してビルドが落ちないようにする。

2. **アイテム抽出**
   - 記事タイトル・本文からキーワード抽出（例: チェア / デスク / モニター / マウス / キーボード / 服 / 靴 / PC / GPU / SSD など）。
   - アイテム名・カテゴリ・要約・画像URL・紹介元URLを整形する。

3. **トップページ構成**
   - 上部: おすすめアイテムの画像が横に並び、**自動横スクロール**するカルーセル。
   - 下部: カテゴリ一覧から各カテゴリ/アイテムに遷移できるナビゲーション。

4. **個別アイテムページ**
   - アイテム詳細・画像・Amazonアソシエイトリンクを掲載。
   - そのアイテムが紹介されていた**記事URLをリスト**で表示。

5. **Amazonアソシエイト**
   - ASINがある場合: `https://www.amazon.co.jp/dp/{ASIN}/?tag=utag-22`
   - ASINがない場合: 検索リンク `https://www.amazon.co.jp/s?k={キーワード}&tag=utag-22`

6. **GitHub Pages公開**
   - GitHub Actions でビルド → `dist/` を `actions/deploy-pages` で公開。
   - `main` ブランチ push / 手動実行 / 定期実行（cron）で更新。

## 実装要件
- 生成物は `dist/` に静的HTML/CSS/JSとして出力する。
- 可能であれば `scripts/generate.py` などの生成スクリプトで再現性を持たせる。
- API取得が失敗した場合は `data/sample_items.json` などのサンプルデータへフォールバック。
- 画像は記事/商品から取得できない場合はフリー素材のプレースホルダーを使用。

## 追加条件
- 既存コードがある場合は最小変更で対応。
- ページ生成後は GitHub Actions がエラーなく完走すること。
- 生成後の `index.html` と `items/*.html` から遷移できること。

以上の条件でサイト生成を完了してください。
